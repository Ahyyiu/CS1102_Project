Result

In The first part of the code extracts a dataset from a zip file, retrieves the file paths and class labels, and organizes them in a DataFrame. It then also provides information about the dataset's size, number of classes, and the count of images in each class. The second part uses the train_test_split function to divide the dataset into training, testing, and validation sets. It then assigns the file paths and class labels to each set and prints the sizes of the sets for reference The third part creates ImageDataGenerator objects for training, validation, and testing. These items set specific parameters for data augmentation, batch size, and other settings. They provide non-augmented validation and testing data and augmented training data in batches from the DataFrame. Using the pre-trained weights and the EfficientNetB5 framework, the model is built and trained in the fourth section. It refines the model by thawing and teaching the last layers. After building the model with appropriate settings, a learning rate scheduler is used to modify the learning rate during training. The trained model will be evaluated on a test set in the following steps, after which the test accuracy and loss will be printed. Plots of training, validation accuracy, and loss of calendars will also be drawn, highlighting the calendars with the best validation accuracy and lowest validation loss. Using appropriate generators, the trained models will be evaluated on the training, validation, and test sets. The evaluation results will be printed along with the accuracy and loss for each set. The trained model will be used to build the predictions in Section 7, and Matplotlib will be used to build and plot the confusion matrix. The confusion matrix displays the number of true positives, true negatives, false positives, and false negatives for each category, providing insight into the model's performance. By using the classification_report function in sklearn. metrics, the results are published to the console as a tabular display of each class metric. Lastly, the code saves the trained model to an HDF5 file, loads the model from the saved file, defines a function to predict and display the class label of an image using the loaded model, and demonstrates the prediction by calling the function with a test image.
